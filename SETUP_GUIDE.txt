✅ STEP 2: Create Firebase Project

2. Click "Go to console"
3. Create new project (name: "PaymentReconciliation" or any name)
4. Select region (any region)

✅ STEP 3: Get Firebase Credentials
────────────────────────────────────
1. In Firebase Console → Project Settings (gear icon)
2. Scroll down to "Your apps" section
3. Click "Add app" → Web (</>) icon
4. Copy the config object

Example config:
{
  apiKey: "AIzaSyD...",
  authDomain: "project.firebaseapp.com",
  projectId: "your-project-id",
  storageBucket: "your-project.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef123456"
}

✅ STEP 4: Update .env File
────────────────────────────
Open: c:\Users\Lenovo\Desktop\AiFitness\.env

Replace with your Firebase credentials:

VITE_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com

✅ STEP 5: Create Firestore Database
─────────────────────────────────────
1. In Firebase Console → Firestore Database (left sidebar)

3. Choose region (same as project)
4. Choose "Start in test mode" (for development)
5. Wait for database creation (~1 minute)

✅ STEP 6: Start Development Server

================================================================================
ASSUMPTIONS EMBEDDED IN CODE
================================================================================

1️⃣ MATCHING DATE TOLERANCE: ±1 day
   └─ Why: Bank batches & settles 1-2 days after transaction

2️⃣ AMOUNT TOLERANCE: ±₹0.01 (1 paise)
   └─ Why: Currency conversion & rounding differences

3️⃣ SETTLEMENT DELAY WINDOW: 2 days max
   └─ Why: Standard industry settlement time

4️⃣ REFUND MATCHING: Original transaction must exist
   └─ Why: Standard financial controls

5️⃣ DUPLICATE DETECTION: Same transaction_id appears 2+ times
   └─ Why: Prevents false positives

6️⃣ TIME BOUNDARIES: First to last day of month
   └─ Why: Allows settlements 2 days into next month

7️⃣ CURRENCY: INR (stored in paise × 100)
   └─ Why: Avoids floating-point precision errors

8️⃣ DATA INTEGRITY: All IDs globally unique
   └─ Why: Enables reliable matching
